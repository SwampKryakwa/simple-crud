# Руководство по развёртыванию и применению
## Развёртывание
Развёртывание производится на Apache HTTP-сервере в совместительстве с применением MySql. Предполагается, что базовая настройка Apache и MySql уже произведена, но, для демонстрации, можно воспользоваться пакетом [XAMPP](https://www.apachefriends.org/).

Все файлы репозитория, за исключением каталога `database`, должны быть размещены в исполняемом каталоге Apache (в случае использования XAMPP - каталог htdocs в установочной директории XAMPP).

Для настройки базы данных MySql, требуется создать базу данных, и импортировать в неё sql-файл `simplecrud.sql`, находящийся в каталоге `database` репозитория.

После импорта базы данных, требуется настроить подключение в PHP-скрипте `conf.php` в каталоге `DBHandling` (экземпляра, уже размещённого в исполняемом каталоге Apache). В классе `DBCredentials` содержатся три константы:
- `LOGIN` - Логин для доступа к базе данных (в случае использования XAMPP - логин по умолчанию уже выставлен),
- `PASSWORD` - Пароль для доступа к базе данных (в случае использования XAMPP - пароль по умолчанию уже выставлен),
- `DBNAME` - Наименование созданной базы данных.

## Применение
Взаимодействие с системой производится с помощью отправления HTTP-запросов. Присутствует 5 операций, поддерживаемых системой:
- Создание задачи
- Просмотр списка задач
- Просмотр отдельной задачи
- Обновление задачи
- Удаление задачи

### Создание задачи
Для создания задачи, требуется отправить POST HTTP-запрос на путь `/tasks/` сервера, где размещена система (в случае использования XAMPP, адрес по умолчанию будет `htp://127.0.0.1/tasks/`). HTTP-запрос должен содержать следующие поля:
- `title` - Название задачи,
- `description` - Описание задачи,
- `status` - Статус задачи.
Все поля являются необходимыми, и не могут быть пустыми/состоять из одних пробелов. В ответ на запрос отправляется JSON с HTTP-статусом (поле `status`) и, в случае успеха, идентификатором созданной задачи (поле `id`), или, в случае провала, сообщением об ошибке (поле `detail`). В случае отсутствия или некорректном содержимом необходимых полей, также добавляется поле `missing_fields`, содержащее список некорректно заполненных/пропущенных полей.

### Просмотр списка задач
Для просмотра списка задач, требуется отправить GET HTTP-запрос на путь `/tasks/` сервера, где размещена система. В ответ на запрос отправляется JSON, содержащий единственное поле - `tasks`. В поле содержится список объектов, каждый из которых представляет отдельную задачу. Объект задачи содержит следующие поля:
- `title` - Название задачи,
- `description` - Описание задачи,
- `status` - Статус задачи.

### Просмотр отдельной задачи
Для просмотра отдельной задачи, требется отправить GET HTTP-запрос на путь `/tasks/{id}/`, где размещена система, заменив `{id}` на идентификатор требуемой задачи. В ответ на запрос отправляется JSON, содержащий HTTP-статус (поле `status`) и, в случае успеха, объект задачи (поле `task`) со следующими полями:
- `title` - Название задачи,
- `description` - Описание задачи,
- `status` - Статус задачи.
В случае провала, добавляется поле `detail`, содержащее сообщение об ошибке.

### Обновление задачи
Для обновления задачи, требуется отправить PUT HTTP-запрос на путь `/tasks/` сервера, где размещена система (в случае использования XAMPP, адрес по умолчанию будет `htp://127.0.0.1/tasks/`). HTTP-запрос должен содержать следующие поля:
- `title` - Название задачи,
- `description` - Описание задачи,
- `status` - Статус задачи.
Все поля являются необходимыми, и не могут быть пустыми/состоять из одних пробелов. В ответ на запрос отправляется JSON с HTTP-статусом (поле `status`) и, в случае провала, сообщением об ошибке (поле `detail`). В случае отсутствия или некорректном содержимом необходимых полей, также добавляется поле `missing_fields`, содержащее список некорректно заполненных/пропущенных полей.

### Просмотр отдельной задачи
Для просмотра отдельной задачи, требется отправить DELETE HTTP-запрос на путь `/tasks/{id}/`, где размещена система, заменив `{id}` на идентификатор требуемой задачи. В ответ на запрос отправляется JSON, содержащий HTTP-статус (поле `status`) и, в случае провала, добавляется поле `detail`, содержащее сообщение об ошибке.
